# AI-Shopping 项目结构说明

本文档记录 AI-Shopping 项目的整体架构和模块划分。

## 项目概览

```
AI-Shopping/
├── AI-Shopping-frontier/           # 前端项目（Vue 3）
│   ├── frontier-user/              # 用户端 - AI-Mart 智能购物平台
│   └── frontier-seller/            # 商家端 - AI-Mart 商家服务平台
├── AI-Shopping-backend_Eureka/     # 后端项目（Spring Cloud 微服务）
│   ├── eureka-server/              # 服务注册中心（端口 8761）
│   ├── gateway-service/            # API网关（端口 8080）- 统一入口
│   ├── auth-service/             # 认证服务（端口 8086）- 用户/商家认证
│   ├── common-api/                 # 统一公共模块（DTO 和 Feign）
│   ├── product-service/            # 商品服务（端口 8081）
│   │   ├── ProductUserController    # 用户端接口：/api/user/product/**
│   │   └── ProductSellerController  # 商家端接口：/api/seller/product/**
│   ├── order-service/              # 订单服务（端口 8082）
│   │   ├── OrderUserController      # 用户端接口：/api/user/order/**
│   │   └── OrderSellerController    # 商家端接口：/api/seller/order/**
│   ├── contact-service/            # 联系人服务（端口 8083）
│   ├── logistics-service/          # 物流服务（端口 8084）
│   └── chat-service/               # AI 聊天服务（端口 8085）
└── AI-Shopping-backend/             # [已弃置] 原单体应用（端口 8080）
```

## 前端模块

### frontier-user（用户端）

- **技术栈**: Vue 3 + Vite + Tailwind CSS
- **职责**: 面向消费者的购物平台
- **核心功能**:
  - AI 智能对话购物
  - 商品浏览与搜索
  - 订单管理
  - 联系人/收货地址管理
  - 物流跟踪

### frontier-seller（商家端）

- **技术栈**: Vue 3 + Vite + Tailwind CSS
- **职责**: 面向商家的管理平台
- **核心功能**:
  - 商品管理（上架、编辑、库存）
  - 订单处理（发货、状态更新）
  - 物流管理

## 后端模块

### 微服务架构

| 服务 | 端口 | 职责 | 依赖 |
|------|------|------|------|
| eureka-server | 8761 | 服务注册与发现中心 | - |
| gateway-service | 8080 | API网关、统一入口、认证鉴权 | Eureka |
| auth-service | 8086 | 用户/商家认证、Sa-Token 管理 | Eureka |
| common-api | - | 统一公共模块（DTO 和 Feign 客户端）| - |
| product-service | 8081 | 商品管理（CRUD、库存）| Eureka |
| order-service | 8082 | 订单管理（下单、状态流转）| Eureka, Redis, common-api |
| contact-service | 8083 | 联系人/收货地址管理 | Eureka |
| logistics-service | 8084 | 物流跟踪、发货管理 | Eureka, common-api |
| chat-service | 8085 | AI 智能对话（通义千问）| Eureka, common-api |

### API 接口拆分

**设计原则**：
- 用户端接口：`/api/user/{service}/**`
- 商家端接口：`/api/seller/{service}/**`
- 内部服务调用：`/internal/{service}/**`（绕过网关）

**商品服务 (product-service)**：
| 端 | 控制器 | 路径 |
|----|--------|------|
| 用户端 | ProductUserController | `/api/user/product/all`, `/api/user/product/{id}`, `/api/user/product/search` |
| 商家端 | ProductSellerController | `/api/seller/product/create`, `/api/seller/product/{id}` |

**订单服务 (order-service)**：
| 端 | 控制器 | 路径 |
|----|--------|------|
| 用户端 | OrderUserController | `/api/user/order/place`, `/api/user/order/list`, `/api/user/order/{id}` |
| 商家端 | OrderSellerController | `/api/seller/order/list`, `/api/seller/order/{id}/ship`, `/api/seller/order/{id}/status` |

**认证服务 (auth-service)**：
| 端 | 路径 |
|----|------|
| 用户端 | `/api/user/auth/login`, `/api/user/auth/register`, `/api/user/auth/info` |
| 商家端 | `/api/seller/auth/login`, `/api/seller/auth/register`, `/api/seller/auth/info` |

### 统一公共模块 (common-api)

| 包路径 | 说明 | 包含内容 |
|--------|------|----------|
| `com.gzasc.aishopping.common.dto.product` | 商品 DTO | `ProductDTO`, `StockDeductRequest` |
| `com.gzasc.aishopping.common.dto.order` | 订单 DTO | `OrderDTO` |
| `com.gzasc.aishopping.common.dto.contact` | 联系人 DTO | `ContactDTO` |
| `com.gzasc.aishopping.common.dto.logistics` | 物流 DTO | `LogisticsRequest` |
| `com.gzasc.aishopping.common.feign.product` | 商品 Feign | `ProductFeignClient` |
| `com.gzasc.aishopping.common.feign.order` | 订单 Feign | `OrderFeignClient` |
| `com.gzasc.aishopping.common.feign.contact` | 联系人 Feign | `ContactFeignClient` |
| `com.gzasc.aishopping.common.feign.logistics` | 物流 Feign | `LogisticsFeignClient` |

### 服务间调用关系

```
┌─────────────────────────────────────────────────────────────┐
│                         common-api                          │
│  ┌──────────────┬──────────────┬──────────────┬──────────┐  │
│  │ product DTO/ │  order DTO/  │ contact DTO/ │logistics │  │
│  │   Feign      │   Feign      │   Feign      │ DTO/Feign│  │
│  └──────────────┴──────────────┴──────────────┴──────────┘  │
└─────────────────────────────────────────────────────────────┘
         ↑               ↑               ↑            ↑
         └───────────────┴───────────────┴────────────┘
                    order-service
                    chat-service
                    logistics-service
```

**调用说明**:
- `order-service` 通过 `common-api` 调用 `product-service` 和 `logistics-service`
- `chat-service` 通过 `common-api` 调用 `product-service`、`order-service`、`contact-service`
- `logistics-service` 通过 `common-api` 接收来自 `order-service` 的调用
