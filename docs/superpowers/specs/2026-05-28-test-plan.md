# AI-Shopping 后端微服务测试计划

> 版本: 1.0
> 日期: 2026-05-28
> 项目: AI-Shopping 智能购物平台

---

## 1. 概述

### 1.1 项目背景

AI-Shopping 是一个基于 Spring Cloud Netflix Eureka 的 AI 智能购物平台，采用微服务架构。后端包含 10 个 Maven 模块，分别为：eureka-server、gateway-service、auth-service、product-service、order-service、contact-service、logistics-service、shop-service、chat-service 和 common-api。

### 1.2 测试目标

- 验证每个微服务 API 端点的功能正确性
- 验证服务间 Feign 调用的连通性和数据一致性
- 验证订单状态机、库存预留等核心业务逻辑
- 验证 Redis Stream 事件驱动的可靠性
- 验证 Sa-Token 认证授权体系
- 验证异常处理和错误响应的一致性
- 验证 API 网关路由和限流机制

### 1.3 测试范围

| 范围 | 包含 | 不包含 |
|------|------|--------|
| 后端 Java 微服务 | 10 个 Maven 模块 | 前端 Vue 应用 |
| 测试类型 | 单元测试、集成测试、API 测试 | 性能测试、安全渗透测试 |
| 数据库 | MySQL 表结构验证 | 大数据量测试 |
| 中间件 | Redis、Eureka | 消息队列压力测试 |

## 2. 测试策略

### 2.1 分层测试策略

```
测试金字塔:
          ┌─────────┐
          │ 端到端   │  ← 核心业务流程（下单、支付、发货）
         ┌┤ 测试     ├┐
        ┌┤├─────────┤├┐
        │││ 集成测试 │││  ← Feign 调用、Redis Stream、数据库操作
        │││         │││
       ┌┤├┤├─────────┤├┤┐
       │││││ 单元测试 │││││  ← Service 层逻辑、状态机、工具类
       │││││         │││││
       └┴┴┴┴─────────┴┴┴┴┘
```

### 2.2 单元测试策略

- **框架**: JUnit 5 + Mockito
- **覆盖目标**: Service 层核心业务逻辑、状态机转换、工具类
- **Mock 策略**: Mapper 层、Feign Client、RedisTemplate 使用 Mock
- **覆盖标准**: 核心业务方法分支覆盖率 > 90%

### 2.3 集成测试策略

- **框架**: Spring Boot Test + H2 (内存数据库)
- **覆盖目标**: Controller → Service → Mapper 全链路
- **数据库**: 使用 H2 模拟 MySQL，Schema 通过 SQL 脚本初始化
- **Mock 策略**: Feign 调用使用 `@MockBean`, Redis 使用嵌入式或 Mock

### 2.4 API 测试策略

- **框架**: MockMvc / WebTestClient
- **覆盖目标**: 每个 REST 端点的 HTTP 方法、状态码、请求/响应格式
- **验证点**: 参数校验、认证鉴权、业务异常、成功响应

## 5. 各服务测试重点

### 5.1 eureka-server
- Eureka 服务注册与发现功能
- WebSecurity Basic 认证配置
- 自我保护模式配置

### 5.2 gateway-service
- IP 限流过滤器 (Caffeine 缓存)
- Sa-Token 认证过滤器 (白名单、Token 校验、角色鉴权)
- 全局异常处理器 (GatewayAuthException、ResponseStatusException)
- 路由规则正确性 (用户端/商家端/内部路由)
- CORS 配置

### 5.3 auth-service
- 用户注册 (唯一性校验、BCrypt 加密)
- 商家注册 (唯一性校验、BCrypt 加密)
- 用户/商家登录 (BCrypt 验证、Sa-Token 登录)
- 登出 (Sa-Token 登出)
- 用户名/手机号查重
- 店员注册 (内部接口)
- 参数校验 (RegisterRequest、LoginRequest)
- BCryptUtil 工具类 (加密、验证、格式校验)

### 5.4 shop-service
- 店铺创建 (自动成为店长)
- 店铺更新 (仅店长可操作)
- 店铺关闭/开启 (软删除)
- 添加店员 (Feign 调用 auth-service)
- 移除店员
- 店铺列表查询 (用户端分页)
- 店铺详情查询 (用户端仅活跃店铺)
- 员工角色查询 (内部接口)
- 店铺信息查询 (内部接口)
- 权限校验 (店长/店员区分)

### 5.5 product-service
- 商品创建 (雪花 ID + 图片)
- 商品更新 (含图片)
- 商品删除 (需下架)
- 商品上架/下架 (salable_products 维护)
- 商品查询 (按 ID、名称、价格范围)
- 商品分页查询 (可售商品)
- 库存扣减 (CAS 乐观锁)
- 库存恢复
- 库存预占 (FOR UPDATE 悲观锁 + 30 分钟过期)
- 预占确认/释放
- 过期预占清理 (定时任务)
- Caffeine 店铺信息缓存

### 5.6 order-service
- 下单 (库存校验 → 订单创建 → 库存预占)
- 取消订单 (PAID 恢复库存 / PENDING 释放预占)
- 支付 (PENDING → PAID → Redis Stream STOCK_CONFIRM)
- 发货 (PAID → SHIPPED → Redis Stream LOGISTICS_CREATE)
- 确认收货 (SHIPPED → DELIVERED)
- 申请退货 (→ RETURN_PENDING)
- 审核退货 (RETURN_PENDING → RETURNING)
- 确认退货 (RETURNING → RETURNED → Redis Stream STOCK_RESTORE)
- 删除订单 (备份到 deleted_orders)
- 订单状态机 (CAS 乐观锁状态转换)
- Redis Stream 消费 (幂等性、兜底文件)
- 超时订单自动取消 (定时任务)
- Redis 订单号生成

### 5.7 contact-service
- 用户联系人创建/删除/更新/列表
- 默认联系人设置
- 店铺地址创建/删除/更新/列表
- 默认发货地址设置
- 联系人查询 (内部接口)
- 所有权校验 (用户联系人、店铺地址)

### 5.8 logistics-service
- 物流记录创建 (发货/退货类型)
- 物流记录查询 (按 ID、订单号、快递单号)
- 最新物流查询 (按订单号+类型)
- 物流记录删除

### 5.9 chat-service
- AI 聊天接口 (ChatController)
- LangChain4j @AiService 配置
- ProductTools (商品查询 Tool)
- OrderTools (订单查询 Tool)
- UserContext (用户 ID 提取)
- Feign 调用链路 (chat → product/order)
- AiResponse 结构化输出
- GlobalExceptionHandler

### 5.10 common-api (作为公共模块引用)
- ApiResponse 结构一致性 (各服务响应格式统一)
- SnowflakeIdGenerator 雪花 ID 生成
- DTO 序列化/反序列化
- Feign 接口定义正确性

## 6. 风险与缓解措施

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Redis Stream 事件在测试中不可靠 | 订单事件无法消费 | 中 | 使用 Mock 模拟 RedisTemplate，单元测试覆盖事件分发逻辑 |
| Feign 服务间调用在单元测试中不可用 | 集成测试覆盖不足 | 高 | 使用 MockBean 模拟 Feign Client |
| AI 模型 (DashScope) 在测试中不可用 | chat-service 无法端到端测试 | 高 | Mock LangChain4j 的 Assistant 接口 |
| H2 与 MySQL 行为差异 | 测试通过但生产有问题 | 低 | 关键 SQL 使用 `@IntegrationTest` 标记 |
| 数据库独立导致跨服务数据一致性 | 分布式事务问题 | 中 | 通过 Feign 调用测试覆盖 |
