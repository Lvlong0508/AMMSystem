# AI-Shopping 后端代码规格说明

本文档适用于 `AI-Shopping-backend_Eureka` 微服务架构项目。

## 1. 技术栈

- **框架**: Spring Boot 3.2.3
- **Java版本**: Java 17
- **微服务框架**: Spring Cloud 2023.0.x
- **服务注册**: Spring Cloud Netflix Eureka
- **服务调用**: OpenFeign
- **持久层**: MyBatis 3.0.3
- **数据库**: MySQL 8.x
- **缓存**: Redis
- **AI集成**: LangChain4j 0.35.0 (阿里云DashScope)
- **构建工具**: Maven

## 3. 代码规范

### 3.1 包命名规范

#### 单体服务包结构
- 根包: `com.gzasc.{服务名}` (如 `com.gzasc.product`, `com.gzasc.order`)
- 子包按功能划分: `controller`, `service`, `mapper`, `model`, `config`, `client`

#### API模块包结构（common-api）
- DTO根包: `com.gzasc.aishopping.common.dto.{服务名}`
  - 例如: `com.gzasc.aishopping.common.dto.product`
  - 包含: `ProductDTO`、`StockDeductRequest` 等 DTO 类
- Feign根包: `com.gzasc.aishopping.common.feign.{服务名}`
  - 例如: `com.gzasc.aishopping.common.feign.product`
  - 包含: `ProductFeignClient` 等 Feign 接口

### 3.2 类命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 实体类 | 名词，驼峰命名 | `Product`, `Order`, `Contact` |
| 控制器 | 名词+Controller | `ProductQueryController`, `OrderUserController` |
| 服务接口 | 名词+Service | `OrderService`, `ProductService` |
| 服务实现 | 名词+ServiceImpl | `OrderServiceImpl`, `ProductServiceImpl` |
| Mapper | 名词+Mapper | `OrderMapper`, `ProductMapper` |
| DTO | 描述性名词 | `PlaceOrderRequest` |
| Feign接口 | 服务名+FeignClient | `ProductFeignClient`, `OrderFeignClient` |
| 启动类 | 服务名+Application | `ProductServiceApplication` |

### 3.3 代码风格

#### 3.3.1 实体类 (Model)

```java
package com.gzasc.product.model;

import lombok.Data;
import lombok.AllArgsConstructor;
import lombok.NoArgsConstructor;

@Data
@NoArgsConstructor
@AllArgsConstructor
public class Product {
    private String id;           // 商品ID
    private String name;         // 商品名称
    private Double price;        // 商品价格
    private String tags;         // 商品标签
    private String description;  // 商品描述
    private Integer stock;       // 商品库存
    private Date createdAt;      // 创建时间
    private Date updatedAt;      // 更新时间
}
```

**规范要点**:
- 使用 Lombok 注解 (`@Data`, `@NoArgsConstructor`, `@AllArgsConstructor`)
- 字段使用驼峰命名
- 字段后添加中文注释说明
- 优先使用包装类型 (`Double`, `Integer`) 而非基本类型

#### 3.3.2 控制器 (Controller)

```java
package com.gzasc.product.controller;

import com.gzasc.product.service.ProductService;
import com.gzasc.product.model.Product;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;
import java.util.List;
import java.util.Map;

/**
 * 商品用户端控制器 - 提供商品查询功能
 * 路径规范：/api/user/{service}/**
 */
@RestController
@RequestMapping("/api/user/product")
@RequiredArgsConstructor
public class ProductUserController {

    private final ProductService productService;

    @GetMapping("/all")
    public Map<String, Object> getAllProducts(
            @RequestParam(name = "page", defaultValue = "0") int page) {
        try {
            List<Product> products = productService.getAllProducts(page);
            return Map.of("message", "查询成功", "data", products, "page", page);
        } catch (Exception e) {
            return Map.of("message", "查询错误：" + e.getMessage());
        }
    }
}

/**
 * 商品商家端控制器 - 提供商品管理功能
 * 路径规范：/api/seller/{service}/**
 */
@RestController
@RequestMapping("/api/seller/product")
@RequiredArgsConstructor
public class ProductSellerController {

    private final ProductService productService;

    @PostMapping("/create")
    public Map<String, String> createProduct(@RequestBody Product product) {
        // 商家创建商品逻辑
    }
}
```

**规范要点**:
- 使用 `@RestController` 注解
- 使用 `@RequiredArgsConstructor` 实现依赖注入
- 控制器按角色拆分：用户端 `/api/user/{service}/**`，商家端 `/api/seller/{service}/**`
- 原路径 `/api/{service}/**` 保留但标记废弃（向后兼容）
- 内部服务调用使用 `/internal/{service}/**`
- 返回类型统一使用 `Map<String, Object>` 或 `Map<String, String>`
- 响应包含 `message` 字段描述操作结果
- 使用 `try-catch` 包裹业务逻辑，统一异常处理
- 使用构造器注入 (`private final`)

#### 3.3.3 服务层 (Service)

**接口定义**:
```java
public interface OrderService {
    /**
     * 创建订单
     * @param order 订单信息
     */
    int createOrder(Order order);
    
    Order getOrderById(String orderId);
    List<Order> getAllOrders();
}
```

**实现类**:
```java
@Service
public class OrderServiceImpl implements OrderService {
    
    @Autowired
    private OrderMapper orderMapper;

    @Override
    public int createOrder(Order order) {
        System.out.println(new Date() + ": run createOrder");
        try {
            return orderMapper.insertOrder(order);
        } catch (Exception e) {
            e.printStackTrace();
            return -1;
        }
    }
}
```

**规范要点**:
- 接口中方法使用 JavaDoc 注释
- 实现类使用 `@Service` 注解
- 方法执行前打印日志，格式: `System.out.println(new Date() + ": run 方法名")`
- 返回 `int` 表示操作影响行数，-1 表示失败

#### 3.3.4 Mapper 接口

```java
@Mapper
public interface ProductMapper {

    @Select("SELECT * FROM products LIMIT 20 OFFSET #{offset}")
    List<Product> selectProductsByPage(@Param("offset") int offset);

    /**
     * 根据商品名称查询商品（模糊查询）
     */
    @Select("SELECT * FROM products WHERE name LIKE CONCAT('%', #{name}, '%')")
    List<Product> selectProductsByName(@Param("name") String name);

    @Insert("INSERT INTO products (id, name, price, tags, description, stock, created_at, updated_at) " +
            "VALUES (#{id}, #{name}, #{price}, #{tags}, #{description}, #{stock}, NOW(), NOW())")
    int insertProduct(Product product);

    @Update("UPDATE products SET stock = stock - #{quantity} WHERE id = #{productId} AND stock >= #{quantity}")
    int deductStock(@Param("productId") String productId, @Param("quantity") int quantity);

    @Delete("DELETE FROM products WHERE id = #{productId}")
    int deleteProduct(@Param("productId") String productId);
}
```

**规范要点**:
- 使用 `@Mapper` 注解
- 使用注解方式编写 SQL (`@Select`, `@Insert`, `@Update`, `@Delete`)
- 复杂 SQL 添加 JavaDoc 注释
- 参数使用 `@Param` 注解
- SQL 中表字段使用下划线命名 (`created_at`)
- 插入/更新使用 `NOW()` 函数设置时间

return Map.of("message", "创建订单错误：" + e.getMessage());

### 6.2 订单ID生成

```java
@Override
public String generateOrderId() {
    // 格式: 年月日(8位) + 序号(5位) + 随机字母(5位)
    // 示例: 2026040100001ABCDE
    String currentDate = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    String key = "order:seq:" + currentDate;
    Long sequence = redisTemplate.opsForValue().increment(key);
    // 设置24小时过期
    if (sequence != null && sequence == 1) {
        redisTemplate.expire(key, 24, TimeUnit.HOURS);
    }
    String seqStr = String.format("%05d", sequence);
    String randomChars = generateRandomLetters();
    return currentDate + seqStr + randomChars;
}
```

## 11. Feign 接口规范（API模块）

### 11.1 Feign 接口定义

```java
package com.gzasc.aishopping.common.feign.product;

import com.gzasc.aishopping.common.dto.product.ProductDTO;
import com.gzasc.aishopping.common.dto.product.StockDeductRequest;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.*;
import java.util.List;
import java.util.Map;

/**
 * 商品服务 Feign 客户端
 * 供其他服务调用商品相关接口
 */
@FeignClient(name = "product-service")
public interface ProductFeignClient {

    /**
     * 获取所有商品（用户端接口）
     */
    @GetMapping("/api/user/product/all")
    Map<String, Object> getAllProducts(@RequestParam("page") int page);

    /**
     * 根据ID查询商品（内部服务调用）
     * 注意：内部调用使用 /internal 前缀，绕过网关认证
     */
    @GetMapping("/internal/product/{productId}")
    ProductDTO getProductById(@PathVariable("productId") String productId);

    /**
     * 扣减库存
     */
    @PostMapping("/internal/product/deduct-stock")
    Map<String, Object> deductStock(@RequestBody StockDeductRequest request);

    /**
     * 恢复库存
     */
    @PostMapping("/internal/product/restore-stock")
    Map<String, Object> restoreStock(@RequestBody StockDeductRequest request);
}
```

**规范要点**:
- 使用 `@FeignClient` 注解，指定 `name`（服务名）和 `path`（基础路径）
- 接口路径与对应 Controller 保持一致
- 参数使用 `@PathVariable`、`@RequestParam`、`@RequestBody` 明确标注
- 复杂请求体使用 `Map<String, Object>` 或 DTO 对象
- 接口需添加 JavaDoc 注释说明用途

## 12. 服务间调用规范

### 12.1 引入依赖

```java
package com.gzasc.aishopping.order.service;

import com.gzasc.aishopping.common.feign.product.ProductFeignClient;
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class OrderServiceImpl implements OrderService {

    private final OrderMapper orderMapper;
    private final ProductFeignClient productFeignClient;  // Feign客户端
}
```

### 12.2 调用示例

```java
import com.gzasc.aishopping.common.dto.product.ProductDTO;
import com.gzasc.aishopping.common.dto.product.StockDeductRequest;

@Override
public int createOrder(Order order) {
    System.out.println(new Date() + ": run createOrder");
    try {
        // 1. 调用商品服务查询商品信息
        ProductDTO product = productFeignClient.getProductById(order.getProductId());
        if (product == null) {
            return -1;
        }

        // 2. 调用商品服务扣减库存
        StockDeductRequest request = new StockDeductRequest(
            order.getProductId(),
            order.getQuantity()
        );
        Map<String, Object> result = productFeignClient.deductStock(request);

        // 3. 创建订单
        return orderMapper.insertOrder(order);
    } catch (Exception e) {
        e.printStackTrace();
        return -1;
    }
}
```

**规范要点**:
- 通过 `common-api` 模块引入 Feign 接口和 DTO
- DTO 位于 `com.gzasc.aishopping.common.dto.{服务名}` 包下
- Feign 客户端位于 `com.gzasc.aishopping.common.feign.{服务名}` 包下
- 外部接口使用 `/api/user/**` 或 `/api/seller/**` 路径
- 内部接口使用 `/internal/**` 路径（绕过网关认证）
- 使用构造器注入 Feign 客户端
- 调用前检查返回结果
- 异常处理与服务内部调用一致

## 13. 配置文件规范

### 13.1 Eureka Server 配置

```yaml
server:
  port: 8761

eureka:
  instance:
    hostname: localhost
  client:
    register-with-eureka: false
    fetch-registry: false
    service-url:
      defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka/
```

### 13.2 微服务配置

```yaml
server:
  port: 8081

spring:
  application:
    name: product-service
  datasource:
    url: jdbc:mysql://localhost:3306/aishopping?serverTimezone=Asia/Shanghai
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver

eureka:
  client:
    service-url:
      defaultZone: http://localhost:8761/eureka/
  instance:
    prefer-ip-address: true

mybatis:
  type-aliases-package: com.gzasc.product.model
  configuration:
    map-underscore-to-camel-case: true
```

**规范要点**:
- `spring.application.name` 必须与 `@FeignClient(name="")` 一致
- `eureka.client.service-url.defaultZone` 指向 Eureka Server
- `prefer-ip-address: true` 使用 IP 注册而非主机名
- 各服务数据库配置独立（可连接同一数据库或分库）
